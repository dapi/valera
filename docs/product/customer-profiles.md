# Профили клиентов автосервиса

**Создан:** 28.12.2025
**Назначение:** Customer Development, генерация тестовых данных, обучение AI
**Статус:** Active

---

## Обзор

Профили клиентов описывают типичные паттерны поведения пользователей AI-ассистента автосервиса. Используются для:

- **Customer Development** — понимание потребностей и болей клиентов
- **AI Training** — генерация реалистичных тестовых диалогов
- **Analytics** — сегментация и анализ конверсии
- **Product Development** — приоритизация функций

---

## Профили

### 1. Активный клиент (20%)

**Характеристика:** Постоянный клиент, доверяет сервису, несколько автомобилей в семье.

| Параметр | Значение |
|----------|----------|
| Booking Rate | 90% |
| Сообщений в диалоге | 10-20 |
| LTV | Высокий |
| Retention | Высокий |

**Поведение:**
- Знает что нужно (ТО, замена масла, диагностика)
- Готов записаться на конкретную дату
- Спрашивает про скидки для постоянных
- Охотно оставляет контактные данные
- Может рекомендовать друзьям

**Типичные автомобили:** Toyota Camry, Kia Optima, BMW X3, Mercedes E-Class

**Сценарий-основа:** [07-ideal-booking-flow.md](../user-scenarios/07-ideal-booking-flow.md)

---

### 2. Разовый клиент (50%)

**Характеристика:** Ищет автосервис для конкретной проблемы, первое обращение.

| Параметр | Значение |
|----------|----------|
| Booking Rate | 50% |
| Сообщений в диалоге | 5-12 |
| LTV | Средний (потенциально высокий) |
| Retention | Зависит от опыта |

**Поведение:**
- Описывает конкретную проблему (вмятина, царапина, стук)
- Интересуется ценой и сроками
- Сравнивает с другими сервисами
- Нуждается в доверии перед решением
- Может стать постоянным при хорошем опыте

**Типичные автомобили:** Hyundai Solaris, Volkswagen Polo, Renault Duster, Kia Rio, Skoda Rapid

**Сценарий-основа:** [01-service-dialog-scenarios.md](../user-scenarios/01-service-dialog-scenarios.md)

---

### 3. "Просто спросить" (20%)

**Характеристика:** Интересуется ценами, не готов к действиям.

| Параметр | Значение |
|----------|----------|
| Booking Rate | 0% |
| Сообщений в диалоге | 3-6 |
| LTV | Нулевой (пока) |
| Retention | N/A |

**Поведение:**
- Задаёт общие вопросы ("сколько стоит покраска?")
- Использует возражения: "дорого", "я подумаю", "нет времени"
- Не называет марку авто или говорит "ещё не купил"
- Не оставляет контактные данные
- Резко заканчивает диалог

**Типичные возражения:**
- "Это дорого"
- "Я подумаю"
- "Нет времени"
- "В другом месте дешевле"

**Типичные автомобили:** Не указывает или "пока выбираю"

**Сценарий-основа:** [05-sales-objection-handling.md](../user-scenarios/05-sales-objection-handling.md)

---

### 4. Срочный клиент (10%)

**Характеристика:** Аварийная ситуация, нужна помощь срочно.

| Параметр | Значение |
|----------|----------|
| Booking Rate | 85% |
| Сообщений в диалоге | 6-10 |
| LTV | Средний-высокий |
| Retention | Высокий (если помогли) |

**Поведение:**
- Описывает аварийную ситуацию (ДТП, поломка)
- Спрашивает "можно ли сегодня/завтра?"
- Готов приехать сразу
- Быстро принимает решения
- Высокая эмоциональность

**Типичные запросы:**
- "Попал в ДТП, нужен срочный ремонт"
- "Поцарапал машину, можно сегодня?"
- "Разбил стекло, когда можете?"

**Типичные автомобили:** Любые, с детальным описанием повреждений

**Сценарий-основа:** Комбинация сценариев + экстренные кейсы

## Использование профилей

### Для Customer Development
- Интервью с реальными клиентами каждого типа
- Валидация гипотез о болях и потребностях
- A/B тестирование коммуникаций

### Для AI/Product
- Генерация тестовых диалогов: [GitHub Issue #132](https://github.com/dapi/valera/issues/132)
- Обучение распознаванию типа клиента
- Персонализация ответов

### Для аналитики
- Сегментация в `user_segment` событиях
- Анализ воронки по типам клиентов
- Оптимизация конверсии
